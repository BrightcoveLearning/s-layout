Introduction

Player E is the latest player, and eventually all new customers automatically get Player E.

Player E is a player for VOD videos, linear channels and live events in which viewers watch media assets. The player incorporates many new features across all platforms. The following two documents discuss and contain videos that demonstrate those features:

  • Player E VOD Features
  • Player E Live Features
In Beacon web apps, all Player E instances are based on Brightcove Player. Beacon automatically uses the Brightcove Default Player for Player E functionality. At this time the player version supported in Beacon is 6.66.8 For more information on the instances and what Brightcove Player plugins are used for each instance, see the Brightcove Player Plugin Compatibility with Player E document.
When using Player E, the viewer is sent back to the previous screen when playback for a movie, a live event, or the last episode of a season ends.

VOD Player selection

To select and configure your player, follow these steps:

  1. In the primary navigation, click Layout.
  2. Click Settings .

  3. The App Settings page appears. In Assets, click Videos to start.

  4. Click Player to see how the player displays information about the currently playing video.
  5. You can change the layouts of the thumbnail, either Landscape or Portrait.
    • Landscape: Landscape images are displayed on the page.
    • Portrait: Portrait images are displayed on the page.

Limitations

Default Audio Track Selection

Default Audio Selection allows you to listen to content in your preferred language based on your location, browser/device settings, etc., without continuously selecting the desired audio track on the player.

General behavior: If you explicitly select a preferred audio track on the Beacon app, that preferred audio track will be used as a reference to auto-select the same audio track language on the following video (if that audio track language is available). This option is saved locally on each device (even on the same user on the same platform on two or more devices; this will be specific on each device).
